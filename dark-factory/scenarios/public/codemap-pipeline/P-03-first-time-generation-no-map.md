# Scenario: P-03 — First-Time Generation — No Map Exists

## Type
feature

## Priority
critical — every new Dark Factory project hits this path on first use; it must work silently without blocking the developer

## Preconditions
- `dark-factory/code-map.md` does NOT exist
- The project has source files (not greenfield — e.g., `.claude/agents/*.md`, `.claude/skills/*/SKILL.md`, `scripts/init-dark-factory.js`, `tests/dark-factory-setup.test.js`)
- The developer has NOT previously run `/df-onboard`
- The developer runs `/df-intake some feature description`

## Action
The df-intake skill executes its Step 0 pre-phase:
1. Attempt to read `dark-factory/code-map.md` — file not found
2. No map exists → invoke codemap-agent for full scan (first-time generation mode)
3. codemap-agent scans all source files, builds complete map
4. codemap-agent writes `dark-factory/code-map.md` with header:
   - `> Auto-generated by Dark Factory codemap-agent. Last analyzed: {ISO date}`
   - `> Git hash: {current HEAD hash}`
   - `> Coverage: FULL`
5. Proceed to Step 1 (spawn spec leads)

## Expected Outcome
- `dark-factory/code-map.md` is created with valid content (all sections present: Module Dependency Graph, Entry Point Traces, Shared Dependency Hotspots, Interface/Contract Boundaries, Cross-Cutting Concerns, Circular Dependencies, Dynamic/Runtime Dependencies)
- The `Git hash:` header line contains the current HEAD hash
- NO developer sign-off prompt is shown (this is auto-generation, not a `/df-onboard` explicit run)
- Spec leads proceed with the newly created map
- The map's `Git hash:` matches `git rev-parse HEAD` so the next pipeline invocation will find a hash match and skip re-scan
- A non-blocking suggestion is shown to the developer: "Code map auto-generated. For a complete, reviewed map run `/df-onboard`."

## Failure Mode (if applicable)
If codemap-agent fails during first-time generation, the map either does not exist or exists with a `COVERAGE: PARTIAL` flag. The pipeline should still proceed — agents use the partial map (or no map) with the caveat noted in the header.

## Notes
This scenario intentionally omits developer sign-off. That is a key behavioral requirement. The sign-off flow is ONLY present during explicit `/df-onboard` runs. Auto-generation triggered by the pre-pipeline hook must be silent.

Contrast with the existing codemap-agent behavior (with sign-off): the full sign-off flow in codemap-agent.md must be conditional — "if invoked from onboard-agent, require sign-off; if invoked from pre-pipeline hook, write immediately."

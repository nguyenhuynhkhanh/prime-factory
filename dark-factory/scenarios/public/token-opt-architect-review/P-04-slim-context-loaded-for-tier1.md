# Scenario: Tier 1 architect reads slim context files by default

## Type
feature

## Priority
high -- slim context loading is the second major token savings mechanism

## Preconditions
- `dark-factory/project-profile-slim.md` exists (generated by token-opt-slim-context)
- `dark-factory/code-map-slim.md` exists (generated by token-opt-slim-context)
- A Tier 1 spec exists and the architect-agent has been updated with slim context loading logic
- The architect-agent is spawned with tier parameter "Tier 1"

## Action
The architect-agent begins its Step 1 (Deep Review) for a Tier 1 spec.

## Expected Outcome
- Architect-agent reads `dark-factory/project-profile-slim.md` (NOT `project-profile.md`)
- Architect-agent reads `dark-factory/code-map-slim.md` (NOT `code-map.md`)
- The architect proceeds with its review using the slim context
- The review is completed without needing to fall back to full files (the slim files contain sufficient context for a 2-file, no-migration spec)
- The full `project-profile.md` and `code-map.md` are NOT read during this session

## Notes
Validates FR-6 (slim context loading for Tier 1). This scenario depends on token-opt-slim-context having run first and generated the slim files. The test validates that the reading behavior changed, not the content of the slim files themselves.

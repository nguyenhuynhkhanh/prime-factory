# Scenario: Agent gracefully handles missing profile sections

## Type
edge-case

## Priority
high -- backward compatibility with profiles generated by old template

## Preconditions
- All 6 consuming agent files exist in `.claude/agents/`
- Each agent has section-targeted profile reading instructions

## Action
Read each consuming agent file. Verify that the section-targeted reading instructions include graceful degradation for missing sections.

## Expected Outcome
- Each agent's profile reading instruction includes language indicating sections may not exist (e.g., "if present", "focus on these sections if they exist", or equivalent)
- No agent's instructions assume all sections WILL be present
- The fallback is to proceed with whatever context IS available, not to error or skip profile reading entirely
- The existing "if it exists" conditional on the profile file itself is preserved (not just section-level graceful degradation but file-level too)
- Section-targeted instructions are ADDITIVE to existing behavior (BR-1) -- agents should not lose access to sections they currently read

## Failure Mode (if applicable)
If section targeting is implemented as "read ONLY these sections," agents could lose context they currently get from reading the whole file. Section targeting should be "focus on" not "read only."

## Notes
This is critical for backward compatibility. Existing profiles lack the new sections. Agents must not break when those sections are absent.
